# 프로젝트 설정

## 자동 활성화 기능

이 프로젝트에서는 다음 기능이 자동으로 활성화됩니다:

### 1. 토큰 모니터링
- 세션 중 주기적으로 남은 토큰 수를 체크합니다
- 작업 완료 시점에 커밋/배포를 위한 충분한 토큰이 남았는지 확인합니다
- 토큰이 30,000 이하로 떨어지면 사용자에게 "커밋/배포하기 좋은 타이밍"임을 알립니다

### 2. 작업 완료 시 알림
- 의미있는 작업이 완료되면 커밋/배포 여부를 확인합니다
- 여유 토큰이 있을 때 미리 알려줍니다

## 개발자 성향

### 코드 설계 철학
- **구조적/근본적 개선 우선**: 빠른 임시 수정보다 구조적이고 근본적인 개선을 선호
- **안정성 중심**: 성능이나 속도보다 안정성과 유지보수성을 우선시
- **급하지 않음**: 시간을 들여서라도 올바른 방향으로 개선하는 것을 선호
- **장기적 관점**: 당장의 문제 해결보다 향후 확장성과 유지보수를 고려한 설계 선호

### 기술적 선택 지침
- 임시방편(quick fix) vs 재설계(refactoring) 선택 시 → **재설계 선택**
- 수정(patch) vs 구조 개선(structural improvement) 선택 시 → **구조 개선 선택**
- 스키마 변경이 필요한 근본적 개선이라도 장기적으로 더 나은 방향이면 → **적극 제안**

## Claude Code에게

**중요**: 이 프로젝트에서 작업할 때는 항상:
1. 세션 시작 시 토큰 모니터링 모드를 활성화하세요
2. 작업이 어느 정도 진행되었을 때 (토큰이 50,000 정도 남았을 때) 사용자에게 커밋/배포 타이밍을 알려주세요
3. 토큰이 30,000 이하로 떨어지면 반드시 경고하세요
4. **문제 해결 시 구조적/근본적 개선 방안을 우선 제안하세요** (사용자는 안정성과 장기적 관점을 중시합니다)
